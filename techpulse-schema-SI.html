03
Transition MLD → NoSQL Firestore
4 règles de re-projection : dénormalisation orientée lecture, sous-collections, duplication contrôlée
Correspondances SQL → Firestore TRANSITION
Les 4 règles de la transition
R1Table → Collection · Ligne → Document JSON autonome identifié par un ID auto-généré Firestore
R2Clé étrangère lue conjointement → Dénormalisation : on embarque le champ directement dans le document (ex : pole: "dev" dans le document candidat)
R3Relation 1-N volumineuse → Sous-collection Firestore (ex : answers/ et choices/) pour ne charger que ce qu'on lit
R4On modélise pour la lecture, pas pour l'écriture. La redondance est acceptée car il n'y a pas de jointures en NoSQL
